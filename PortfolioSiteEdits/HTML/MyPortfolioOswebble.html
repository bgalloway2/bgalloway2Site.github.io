Oswebble: An Interactive Scrabble-Based Game

Oswebble a Scrabble inspired mobile web application game. A group of developers and designers (students) were tasked to build an Oswego themed game for SUNY Oswego for on campus recreation. I acted as team leader to the GUI team, tasked with creating both a mobile app interface and The work was done using an agile methodology with weekly sprints. an interface for the kiosk screen users would interact with. The goals of our team were to ensure we met every requirement that project manager gave us, so users could enjoy a smoothexperience.

Our Teams Goal

My team was expected to work closely with requirements to ensure that our interface accounted for every feature the project manager requested. From this, we were tasked with creating a number of wireframes for each of these features. We were then expected to design a prototype based off of these for user-testing. The final steps were to develop an interface screen for both the mobile application users would interact with, and the kiosk screen that displayed the moved of each player in the game.

Design

Wireframing Prototyping Development

My specific role for my team was to create wireframes and work side by side with a usability team to ensure that our design was user-friendly. I then was expected to design the prototype that we made. As team leader, I was expected to manage the development work of the other members on my team.



User flow

This diagram showed the flow of thes system from a user perspective. The user was expected to land on the login page for the application. The expectationw as for the user to enter their information, and proceed to the game board. When an endgame condition was met, the user would choose whether to play again and return to the game board, or to exit the application



Wireframing

The design goal was to keep visual selectors minimal, having fewer functions made for an interface that was easier for a user to learn and interact with.

Design: To place a tile on the board, click on a tile, then click a place on the board to move the tile.

Design: Users were also given 4 buttons on the bottom to interact with their tile hand, as well as a quit button and help screen button in the top corners of the application.




Final Application

Our end design for the mobile app can be seen on the right, and the kiosk interaction screen below. Following the color scheme guidelines given to us from usability, and matching each feature that requirements had asked for our design proved to be successful.





Project Results

Overall our team and all involved teams were able to integrate multiple parts of a system together to create a fully functioning game that allowed users to connect and drop from an online scrabble game. The application featured AI that would take the place when uniuque users werent present. The logic on the board was accurate and prevented players from making moves that broke the game rules.
